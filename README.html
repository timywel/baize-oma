对应 .md 源文件: README.md (AI 版, 6KB)

当前版本: 0.2.0 (2026-06-25, Phase 1-4 仓内部分完成)

类型: HTTP slot, 端口 20060 (可经 BAIZE_OMA_PORT 环境变量覆盖)

1. 仓定位

baize-oma 是 baize-loop 的 OMA Agent 调度子系统，独立 HTTP 进程。对外暴露 3 个 capability:

  • task.decompose — 任务拆解为 DAG
  • chat.agent.team.schedule — 多 Agent 团队协作
  • chat.loop.execute — 循环执行 (decompose → execute → reflect)

LLM 请求走 baize-switch (端口 20030, 路径 /v1/messages)。

2. 角色 (Role)

type: "http" slot — 独立 Express 进程, 不传染协议给主控。

baize-loop 主控通过 HttpSlotAdapter 跨进程 HTTP 调用本服务。

6. 架构

┌─────────────────────┐         HTTP (port 20060)         ┌──────────────────────────────────┐
│ baize-loop (主控)   │ ────────────────────────────────> │ baize-oma (本仓, HTTP server)    │
│ HttpSlotAdapter     │   POST /chat.agent.*              │   src/server.ts (Express)        │
│                     │   GET  /health /manifest          │   src/routes/* (5 routers)       │
└─────────────────────┘                                    │   src/decomposer/ (decompose)    │
                                                            │   src/dag/ (executor + visualizer) │
                                                            │   src/llm/client.ts (→ 20030)    │
                                                            │   vendor/open-multi-agent/ (1.8.0)│
                                                            └──────────────────────────────────┘

LLM 请求链: baize-oma → baize-switch (20030) → 上游 Anthropic/OpenAI provider。本服务不直连 LLM provider。

8. vendored OMA + patch-package 工作流

OMA 源码 vendored 在 vendor/open-multi-agent/。修改 OMA 内部逻辑的标准流程:

# 1. 修改 vendor/open-multi-agent/dist/... 里某个文件
# 2. 生成 patch
npx patch-package @open-multi-agent/core
# → 生成 patches/@open-multi-agent+core+1.8.0.patch

# 3. 上游升级
npm install @open-multi-agent/core@1.9.0
# → postinstall 自动重打 patch (如果冲突会报错, 手动 rebase)

为什么不直接 fork OMA?

  • 保留升级路径 (上游发版后 npm update 一行升级)
  • patch 文件是 diff，可读可审计
  • 协议上仍只附 OMA 的 MIT LICENSE (vendored + patch 不构成衍生作品)
